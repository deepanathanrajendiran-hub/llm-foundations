Positional Encoding & Long Context

Attention is order-blind, so position must be injected — and how you inject it decides whether your model handles 2K tokens or 2M. Sinusoidal → learned → relative → RoPE → ALiBi, plus the context-extension tricks.

📋 What you’ll learn (~14 min)

Why position is needed (permutation-equivariance) · sinusoidal & learned absolute · relative / T5 bias · RoPE and why it’s relative · ALiBi · NoPE · context extension (PI / NTK / YaRN / LongRoPE) · lost-in-the-middle & attention sinks.

1. Intuition

Self-attention is permutation-equivariant: shuffle the input tokens and the outputs just shuffle the same way — the mechanism literally cannot tell “dog bites man” from “man bites dog.” To it, a sentence is an unordered bag of words. So we inject a position signal — a timestamp on each token. The how matters enormously: it determines whether a model trained at 4K tokens falls apart at 32K, or extends gracefully to millions.

2. How it works — three places to inject position

The original Transformer adds a fixed sinusoidal vector to each token embedding. Modern models instead rotate the query/key vectors (RoPE) or bias the attention scores directly (ALiBi). Where the signal enters changes its behavior — especially how well it extrapolates past the trained length.

Absolute→embedding⊕ position vector→into block
RoPE→Q, K⟳ rotate by position→score depends on m−n
ALiBi→scores⊕ −m·(i−j) bias→softmax
Figure 4 — Add to embeddings (absolute), rotate Q/K (RoPE), or bias the scores (ALiBi).

3. The approaches — full landscape

Sinusoidal absolute classic

What: the original (2017): add fixed sine/cosine vectors of geometrically increasing wavelength to embeddings. How: dimension 2i uses sin(pos/10000^{2i/d}). Solves: gives order for free (no parameters); in theory a fixed linear map relates any two offsets. Cost: extrapolates to longer-than-trained lengths poorly in practice.

Read more: The positional encoding, explained · Kazemnejad

Learned absolute classic

What: a trainable embedding per position 0…L−1 (BERT, GPT-1/2). How: just another lookup table added to token embeddings. Solves: flexible, data-fit positions. Cost: a hard wall at the max trained length L — positions beyond it have no embedding at all, so extending context means adding+training new rows.

Read more: Positional embeddings in Transformers · ICLR 2025 Blog

Relative position (Shaw) & T5 bias relative

What: encode the distance between tokens, not absolute slots. Shaw 2018 adds learned vectors keyed by relative offset into the attention computation; T5 simplifies this to a learned scalar bias per (head, bucketed distance) added to the logits. Solves: translation invariance and cleaner length behavior than absolute. Cost: extra logic/params in the attention step.

Read more: Relative positional encoding · Jake Tae

RoPE — Rotary Position Embedding dominant default

What: rotate Q and K by an angle proportional to position (Su et al., RoFormer 2021) — used by LLaMA, Qwen, Mistral, GPT-NeoX. How: treat dimensions in pairs; rotate pair i by pos·θ_i with θ_i = base^{−2i/d} (base often 10000). Because a dot product of two rotated vectors depends only on the difference of angles, the attention score depends on the relative distance m−n — even though each token is rotated by its own absolute position. Applied to Q/K only (not V). Solves: relative-position behavior with zero added parameters and clean long-context extension.

Pros
  • Relative & absolute in one, parameter-free
  • Extendable (PI/NTK/YaRN below)
Cons
  • Naive extrapolation past training length breaks
  • Interleaved vs half-split conventions trip weight ports

Read more: Rotary Embeddings: A Relative Revolution · EleutherAI

ALiBi — Attention with Linear Biases extrapolation

What: no embeddings at all — add a distance penalty straight to the scores (Press, 2021). How: subtract m·(i−j) from each logit, with a fixed per-head slope m (a geometric sequence across heads). Solves: strong length extrapolation (“train short, test long”) and a built-in recency bias. Cost: the recency bias can hurt tasks needing far-back lookups; less expressive than RoPE.

Read more: ALiBi: attention with linear biases · Ofir Press · GitHub

NoPE — no positional encoding surprising

What: a causal decoder with no explicit position signal (Kazemnejad, 2023). How: the causal mask itself breaks permutation symmetry, so the model can learn position implicitly. Solves: sometimes matches or beats explicit schemes on length generalization — a useful reminder that “you must add position” is not absolute for decoders.

Read more: No Positional Embeddings (NoPE) · Sebastian Raschka

Context-window extension: PI → NTK → YaRN → LongRoPE frontier

What: stretch a RoPE model trained at e.g. 4K to 32K–2M tokens. How: Position Interpolation (Chen 2023) linearly squeezes position indices into the trained range (needs a short fine-tune). NTK-aware scaling changes the base frequency instead, preserving high-frequency (short-range) detail. YaRN (Peng 2023) interpolates frequency bands non-uniformly (“NTK-by-parts”) and adds an attention-temperature correction, reaching long context with far less fine-tuning. LongRoPE (2024) evolutionary-searches per-dimension scales to hit 2M tokens. Solves: long context without retraining from scratch; trade a little short-context quality for a lot of reach.

Read more: Extending RoPE: PI, NTK & YaRN · EleutherAI

5. The math

Sinusoidal positional encoding

$$ PE_{(pos,\,2i)} = \sin\!\left(\frac{pos}{10000^{2i/d}}\right),\qquad PE_{(pos,\,2i+1)} = \cos\!\left(\frac{pos}{10000^{2i/d}}\right) $$

pos = position index, i = dimension pair. Low dimensions oscillate fast, high dimensions slowly — a multi-scale “binary clock.” Geometric wavelengths let a fixed linear map shift any position by a constant offset.

RoPE — rotary embedding (2-D pair)

$$ R(m\theta) = \begin{pmatrix} \cos m\theta & -\sin m\theta \\ \sin m\theta & \cos m\theta \end{pmatrix},\qquad \theta_i = 10000^{-2i/d} $$

The key identity — rotated query at \(m\) dotted with rotated key at \(n\) depends only on \(m-n\):

$$ \big(R(m\theta)\,q\big)^{\top}\big(R(n\theta)\,k\big) = q^{\top} R\big((n-m)\theta\big)\, k $$

Because \(R(a)^{\top}R(b)=R(b-a)\). So an absolute rotation per token yields a score that is purely a function of relative distance \(m-n\) — the magic of RoPE. Position Interpolation replaces \(m\) with \(m\cdot \tfrac{L_{\text{train}}}{L_{\text{target}}}\); NTK/YaRN instead rescale the base inside \(\theta_i\).

ALiBi bias

$$ \text{score}_{ij} = \frac{q_i \cdot k_j}{\sqrt{d_k}} - m\,(i - j) $$

m = fixed per-head slope (a geometric sequence over heads). The penalty grows linearly with distance \(i-j\), encoding recency without any embedding — and it keeps working past the trained length.

6. In code — applying RoPE

python
def apply_rope(x, positions, theta=10000.0):
    # x: (batch, heads, seq, d_head) with d_head even
    b, h, n, d = x.shape
    half = d // 2
    freqs = theta ** (-torch.arange(0, half).float() / half)   # (half,)
    ang = positions[:, None].float() * freqs[None, :]          # (seq, half)
    cos, sin = ang.cos(), ang.sin()
    x1, x2 = x[..., :half], x[..., half:]
    # rotate each (x1, x2) pair by its position-dependent angle
    return torch.cat([x1 * cos - x2 * sin,
                      x2 * cos + x1 * sin], dim=-1)

7. Worked example — why RoPE is “relative”

Work in one 2-D pair with step angle \(\theta = 0.5\) rad/position. Put a query at position \(m=3\) and a key at \(n=1\). RoPE rotates the query by \(m\theta = 1.5\) rad and the key by \(n\theta = 0.5\) rad. Their dot product is

$$ \big(R(1.5)\,q\big)^{\top}\big(R(0.5)\,k\big) = q^{\top} R(0.5 - 1.5)\,k = q^{\top} R(-1.0)\,k. $$

The score depends only on the rotation \(R(-1.0)\) — i.e. the gap of 2 positions, not the absolute slots 3 and 1. Move the same pair to positions 7 and 5 (gap still 2) and you get the identical score. That is exactly the relative-position property, achieved with no learned parameters and nothing added to the values — which is why RoPE became the default.

8. When to use what — decision guide

  • New decoder-only LLM → RoPE (set a larger base θ if you plan long context).
  • Must extrapolate far beyond training cheaply → ALiBi, or RoPE + YaRN/LongRoPE.
  • Extending an existing RoPE model 4K→32K with a small fine-tune budget → Position Interpolation or YaRN; validate on needle-in-a-haystack and multi-hop tasks, and watch short-context regression.
  • Long-context serving → remember the cost is also the KV cache (see Attention): pair your positional scheme with GQA/MLA + FlashAttention.

9. Common pitfalls & misconceptions

  • “Just feed a RoPE model longer sequences.” Naive extrapolation past the trained length collapses; you need PI/NTK/YaRN/LongRoPE.
  • “Bigger context window = the model uses it.” Advertised ≠ effective length. Watch lost-in-the-middle (U-shaped recall: info at the start/end is used far better than the middle) and validate beyond single-needle tests.
  • “Without positional encoding a Transformer is useless.” A causal decoder (NoPE) can learn order from the mask alone — sometimes competitively.
  • “Attention sinks are a bug.” Models dump attention on the first tokens because softmax must sum to 1; StreamingLLM exploits this to stream over millions of tokens — but naively evicting those first tokens breaks generation.
  • “Low perplexity at long context = good long-context model.” Perplexity can stay low while retrieval accuracy collapses; measure effective context directly.

10. 🚀 What’s new in the industry

🚀 Context to millions of tokens
  • RoPE scaling matured. YaRN and LongRoPE stretch RoPE to 128K–2M tokens with modest fine-tuning, balancing long reach against short-context quality.
  • Rethinking position itself. CoPE makes position content-dependent (count what matters), and “Round and Round We Go” shows models use the full RoPE frequency spectrum — not just distance decay.
  • Streaming & sinks. Dedicated sink tokens + sliding windows (StreamingLLM) enable effectively unbounded generation, now common in serving stacks.

12. Key takeaways

  • Attention is order-blind (permutation-equivariant), so position must be injected.
  • RoPE is today’s default — rotating Q/K makes scores depend on relative distance for free, and it extends (PI/YaRN/LongRoPE).
  • ALiBi trades expressiveness for strong, simple length extrapolation; NoPE shows causal decoders can learn order from the mask.
  • Long context is two problems: the positional scheme (this page) and the KV-cache cost (the Attention page).
  • Advertised ≠ effective length: watch lost-in-the-middle and attention sinks; measure recall, not just perplexity.
