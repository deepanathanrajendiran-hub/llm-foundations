Transformer Foundations — Overview

The complete stack that turns text into predictions — tokenization → embeddings → attention → the Transformer block → positional encoding. This overview gives you the whole picture in ~10 minutes; the four deep-dive pages then take each piece classic → SOTA. It’s the bedrock every other LLM topic builds on.

📋 This module, in four parts (~1 hr total)
  1. Tokenization & Embeddings — how raw text becomes a list of integer IDs, and how IDs become vectors. (Module 02)
  2. Attention Mechanisms — the soft content-based lookup at the heart of every block, and its efficient variants. (Module 04)
  3. Transformer Architecture — how attention + FFN + residual + norm stack into a model; BERT vs GPT vs T5. (Module 03)
  4. Positional Encoding & Long Context — how order is injected, and how models reach 100K+ tokens. (Module 05)
Six ideas to land across the module:
  1. Subword tokenization beats both words and characters by trading vocabulary size against sequence length.
  2. Embeddings turn discrete IDs into geometry; context (attention) is what disambiguates meaning.
  3. Attention is a soft, content-based lookup: softmax(QKᵀ/√d)·V.
  4. A Transformer block = attention (mix across tokens) + FFN (think per token), wrapped in residual + norm.
  5. Attention is order-blind, so position must be injected — and how you inject it decides long-context behavior.
  6. Almost every “modern” trick (GQA, FlashAttention, RoPE scaling) exists to fight the n² and KV-cache costs.

1. Why it matters / where it fits

Every large language model is the same four-layer sandwich: (1) a tokenizer chops text into a fixed vocabulary of pieces, (2) an embedding table maps each piece to a vector, (3) a stack of Transformer blocks — built around attention — mixes those vectors using context, with positional encoding telling the model what order things came in, and (4) an output layer turns the final vectors back into a probability over the next token.

These four topics are inseparable in practice and constantly cross-examined in interviews, because almost every downstream concept reduces to them. RAG is “what extra tokens do I put in the context window?” Inference optimization is “how do I make attention and the KV cache cheaper?” Long-context models live or die by their positional scheme. Fine-tuning, MoE, quantization — all operate on this skeleton. If you can draw the pipeline below from memory and explain why each choice was made, you can reason about the rest of the field.

2. Intuition

Tokenization is like writing with LEGO bricks instead of whole pre-built houses. Whole-word “houses” mean you need a brick for every word that exists (millions, and you still miss new ones). Single-letter bricks are universal but you need hundreds of them to build anything. Subwords are the sweet spot: a few common chunks (token, ##ization, ▁the) compose into any word, even ones never seen in training.

Embeddings turn each brick into a point in a high-dimensional “map of meaning,” where nearness encodes similarity: cat and kitten land close together, cat and thermodynamics far apart. But a static map can’t tell which sense of bank you mean. That’s the job of attention.

Attention is a spotlight with a search engine attached. To understand the word it in “the trophy didn’t fit in the suitcase because it was too big,” the model lets it issue a query (“what am I referring to?”), every other word advertises a key (“I’m a noun, I’m the subject…”), and the words whose keys best match the query get the most spotlight — their values are blended into the new representation of it. The Transformer just stacks this “look around and update” step many times. Positional encoding is the timestamp on each word, because the spotlight mechanism, on its own, is blind to order — to it, a sentence is an unordered bag of words.

3. How it all fits — one forward pass

Follow one sentence through the machine; each step is unpacked on its own page:

  • Text → tokens → IDs. A deterministic tokenizer normalizes, splits, applies a subword model (BPE/WordPiece/Unigram), and emits a list of integers — the only thing the network ever sees. → Tokenization
  • IDs → vectors. An embedding matrix of shape (V, d) maps each ID to a learned vector; similar tokens land near each other. → Embeddings
  • Inject position. Because attention is order-blind, a position signal is added (sinusoidal), rotated in (RoPE), or biased into the scores (ALiBi). → Positional Encoding
  • Attention mixes tokens. Each token forms a query/key/value; softmax(QKᵀ/√d)·V blends in the most relevant other tokens. Multi-head runs several in parallel; a causal mask hides the future. → Attention
  • The block thinks, and stacks. A position-wise FFN processes each token, residual + norm wrap both sub-layers, and L blocks stack before a final norm + unembed → next-token probabilities. → Transformers
