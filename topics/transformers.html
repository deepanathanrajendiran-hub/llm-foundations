Transformer Architecture

How attention, a feed-forward network, residual connections, and normalization stack into the model that ate NLP — plus the design choices (pre-norm, RMSNorm, SwiGLU) and families (BERT / GPT / T5) you’ll be asked about.

📋 What you’ll learn (~13 min)

The block (attention + FFN + residual + norm) · pre-norm vs post-norm · RMSNorm & SwiGLU · the residual stream · encoder-only vs decoder-only vs encoder-decoder · parameter counting · depth vs width & Chinchilla scaling.

1. Intuition

A Transformer block does two things: attention lets tokens talk to each other (mix information across the sequence), and a feed-forward network lets each token think on its own (transform its own vector). Wrap both in a residual connection (a gradient highway) and a normalization step (to keep activations well-scaled), stack the block L times, and you have a model that — unlike an RNN — trains fully in parallel and connects any two tokens in one step.

2. How it works — the block, and stacking

A position-wise feed-forward network (FFN) — two linear layers with a nonlinearity, applied identically and independently to every position — expands to ~4× width and back, doing most of the model’s per-token computation (and holding ~⅔ of its parameters). Both sub-layers are wrapped in a residual connection (x + sublayer(x), which creates the “residual stream” gradients flow through) and normalization. Modern models put the norm inside the residual branch (“pre-norm”) for stable deep training. Stack L blocks, apply a final norm, and project back to vocabulary logits. A decoder (GPT) uses causal masking; an encoder (BERT) is bidirectional; an encoder-decoder (T5) adds a cross-attention sub-layer linking the two.

input hidden states  x → RMSNorm → Causal Multi-Head Attention → ⊕ add residual (x + attn) → RMSNorm → Feed-Forward Network  (SwiGLU) → ⊕ add residual → output  →  next block
repeat × L layers, then final RMSNorm → unembed → softmax

3. The approaches — families & block design

Encoder-only (BERT) understanding

What: bidirectional encoder, pretrained with masked-language-modeling. Solves: rich bidirectional representations for classification, NER, retrieval. Cost: can’t generate open-ended text (sees the whole input at once, no left-to-right factorization).

Read more: The Illustrated BERT · Jay Alammar

Decoder-only (GPT) dominant for generation

What: causal decoder, pretrained with next-token prediction. Solves: generation + in-context learning; one simple objective scales beautifully (GPT-2/3 → today). The default for modern LLMs.

Read more: The Illustrated GPT-2 · Jay Alammar

Encoder-decoder (T5) seq2seq

What: encoder reads input, decoder generates output, joined by cross-attention; everything framed as text-to-text. Solves: translation/summarization where a clean input→output split helps. Cost: more complex; for general chat, decoder-only won.

Read more: T5 (encoder-decoder) docs · Hugging Face

Block-design choices: norm, activation, residual modern defaults

Pre-norm vs post-norm: moving LayerNorm inside the residual branch (pre-norm) makes deep stacks train stably without heavy warmup — now standard, with a final norm added at the end. RMSNorm drops mean-centering and bias for a cheaper, equally good norm (LLaMA-family). Activations evolved ReLU → GELU → SwiGLU (a gated FFN that boosts quality, with hidden width scaled to ~⅔·4d to match parameters). Residual stream: the additive highway through all layers — the conceptual backbone of mechanistic interpretability.

Read more: The Big LLM Architecture Comparison · Sebastian Raschka

5. The math

LayerNorm vs RMSNorm

$$ \mathrm{LN}(x) = \gamma\odot\frac{x-\mu}{\sqrt{\sigma^2+\epsilon}}+\beta, \qquad \mathrm{RMSNorm}(x) = \gamma\odot\frac{x}{\sqrt{\tfrac{1}{d}\sum_i x_i^2 + \epsilon}} $$

\(\mu,\sigma^2\) = mean/variance over the feature dim; \(\gamma,\beta\) = learned scale/shift; \(\odot\) = elementwise. RMSNorm drops the mean subtraction and bias \(\beta\) — cheaper, and empirically just as good.

FFN and SwiGLU

$$ \mathrm{FFN}(x) = W_2\,\phi(W_1 x + b_1) + b_2, \qquad \mathrm{SwiGLU}(x) = \big(\,\mathrm{Swish}(W_1 x)\odot (W_3 x)\,\big)W_2 $$

The classic FFN expands to \(d_{ff}\approx 4d\) then back. SwiGLU is a gated variant using three matrices; to keep parameters comparable its hidden width is set to ~\(\tfrac{2}{3}\cdot 4d\).

Parameter counts (interview favorite)

$$ P_{\text{attn}} = 4d^2,\qquad P_{\text{ffn}} = 2\cdot d\cdot d_{ff} = 8d^2,\qquad P_{\text{block}} \approx 12d^2,\qquad P_{\text{model}} \approx 12\,L\,d^2 $$

So the FFN holds \(8d^2/12d^2 = \tfrac{2}{3}\) of a block’s parameters, and a whole model (ignoring embeddings) is ≈ \(12\,L\,d^2\) for \(L\) layers — the back-of-envelope every interviewer wants.

6. In code

A pre-norm block — two residual sub-layers, norm inside each branch:

python
def transformer_block(x, attn, ffn, norm1, norm2):
    # pre-norm: normalize INSIDE the residual branch, then add back
    x = x + attn(norm1(x))     # mix across tokens (causal multi-head attention)
    x = x + ffn(norm2(x))      # think per token (e.g. SwiGLU feed-forward)
    return x

The whole decoder-only forward pass, end to end:

python
def gpt_forward(ids, embed, pos, blocks, final_norm, unembed):
    x = embed(ids) + pos(ids)          # tokens + positions   -> (b, n, d)
    for block in blocks:               # L stacked pre-norm blocks
        x = block(x)
    x = final_norm(x)                  # final RMSNorm
    return x @ unembed                 # logits               -> (b, n, vocab)

7. Worked example — count the parameters

Take a BERT-base-ish config: d = 768, L = 12 layers, FFN ratio 4, vocab V = 30k.

  • Attention per block: \(4d^2 = 4 \times 768^2 \approx 2.36\text{M}\)
  • FFN per block: \(8d^2 = 8 \times 768^2 \approx 4.72\text{M}\) — about twice the attention, the ~⅔ share.
  • Per block: ≈ 7.08M → × 12 layers ≈ 85M.
  • Embedding / unembedding: \(V \cdot d = 30000 \times 768 \approx 23\text{M}\) (tied → counted once).
  • Total ≈ 85M + 23M ≈ 108M — right in the BERT-base ballpark (~110M). The 12 L d² rule nailed it.

8. When to use what — decision guide

  • Generation / chat → decoder-only (GPT/LLaMA). It’s the default.
  • Classification / embeddings / retrieval → encoder (BERT-style): bidirectional context, cheap inference.
  • Translation / summarization with a clean input→output split → encoder-decoder (T5).
  • Depth vs width at a fixed budget: deeper helps reasoning/composition up to a point but is harder to train (needs pre-norm, careful init, residual scaling); wider is easier to parallelize. Most modern LLMs grow both, guided by Chinchilla (~20 training tokens per parameter).

9. Common pitfalls & misconceptions

  • “The FFN is a minor add-on.” It holds ~⅔ of each block’s parameters and does most per-token computation.
  • “Remove the residuals, it’ll be fine.” A deep stack without residual connections won’t train — gradients can’t flow; the residual stream is essential.
  • “Pre-norm vs post-norm is a detail.” Post-norm needs careful warmup and is unstable when deep; pre-norm (norm inside the branch, plus a final norm) is why we can train 100-layer models.
  • “Bigger model is always better.” Chinchilla showed many big models were undertrained; for a fixed compute budget, a smaller model on more tokens often wins.
  • “A decoder can’t classify.” It can (via prompting), but a fine-tuned encoder is usually cheaper and stronger for pure classification.

10. 🚀 What’s new in the industry

🚀 How the block is evolving
  • Modern defaults converged. Pre-norm + RMSNorm + SwiGLU + RoPE + GQA is the de-facto recipe (LLaMA 3, DeepSeek-V3, Qwen).
  • Sparse scaling (MoE). Switch Transformers → DeepSeek-V3 route each token to a few experts, scaling parameters to hundreds of billions while keeping per-token compute roughly constant.
  • Beyond attention. Mamba (selective SSMs) and hybrids like Jamba interleave linear-time recurrence with attention for cheap long context; nGPT rethinks normalization on the hypersphere for faster training.

12. Key takeaways

  • A block = mix (attention) + think (FFN), in a residual + norm wrapper. Stack L, finish with a norm and an unembed.
  • The FFN holds ~⅔ of the parameters; a model is ≈ 12·L·d² ignoring embeddings.
  • Pre-norm + RMSNorm + SwiGLU is the modern recipe; pre-norm is what makes deep stacks trainable.
  • Decoder-only dominates generation; encoders for understanding, encoder-decoders for seq2seq.
  • Chinchilla reset pretraining: balance parameters against ~20× tokens, don’t just grow the model.
