Attention Mechanisms

The soft, content-based lookup at the heart of every Transformer block — from scaled dot-product attention to the efficiency variants (GQA, MLA, FlashAttention, sparse/linear) that make modern LLMs affordable.

📋 What you’ll learn (~15 min)

Q/K/V and scaled dot-product · why the √dk matters · multi-head & causal masking · the KV cache and MQA/GQA/MLA · FlashAttention · sliding-window / sparse / linear attention · attention sinks · a live heatmap demo · the math and the n²/KV-cache cost story.

1. Intuition

Attention is a spotlight with a search engine attached. To understand the word it in “the trophy didn’t fit in the suitcase because it was too big,” the model lets it issue a query (“what am I referring to?”), every other word advertises a key (“I’m a noun, I’m the subject…”), and the words whose keys best match the query get the most spotlight — their values are blended into the new representation of it. Unlike an RNN, this gives an O(1) path between any two tokens and is fully parallel at training time.

2. How it works

From each token’s hidden vector we compute three projections via learned matrices: a query Q = xW_Q, a key K = xW_K, and a value V = xW_V. Then, for every pair of tokens, we score how much token i’s query matches token j’s key with a dot product, scale it, softmax across j so the weights for each query sum to 1, and use those weights to take a weighted average of the values:

x(n × d) → Q, K, V= xW_Q, xW_K, xW_V → scores = QKᵀ/√d(n × n) → + maskcausal / padding → softmaxrows sum to 1 → × V → out · W_O
Figure 2 — Scaled dot-product attention. The only quadratic part is the n × n score matrix.

Multi-head attention runs h of these in parallel on d/h-dimensional slices, so different heads can specialize (one tracks the previous token, one tracks the subject noun, one tracks matching brackets…), then concatenates the results and mixes them with an output projection W_O. For autoregressive (GPT-style) generation we apply a causal mask so token i can only attend to tokens ≤ i — the future is hidden. 3. The approaches — full landscape

Attention from the 2017 core through the efficiency variants every modern LLM uses, classic → SOTA.

RNN / LSTM sequence models what attention replaced

What: the pre-2017 baseline. How: process tokens one at a time, carrying a hidden state. Why we left: strictly sequential (can’t parallelize over time), and the path between distant tokens is O(n), so long-range gradients vanish. Attention makes that path O(1) and fully parallel at training time.

Read more: Understanding LSTM Networks · Christopher Olah

Scaled dot-product self-attention core

What: the 2017 mechanism, softmax(QKᵀ/√d_k)V. How: content-based soft lookup over all positions (§2). Solves: O(1) path length, full parallelism, dynamic context-dependent mixing. Variants of use: self-attention (Q,K,V from the same sequence), cross-attention (queries from the decoder, keys/values from the encoder), and causal/masked attention (future hidden for generation).

Pros
  • Expressive, parallel, general
  • Exact, no approximation
Cons
  • O(n²) compute & memory in sequence length
  • KV cache grows with context at inference

Read more: The Illustrated Transformer · Jay Alammar

Multi-Head Attention (MHA) core

What: h attention heads on d/h-dim subspaces. How: split, attend per head in parallel, concat, project with W_O. Solves: lets the layer attend to multiple relationships at once (syntax, coreference, position) without extra parameters vs one big head. Trade-off: at inference each head keeps its own K and V → fat KV cache, which the next three variants attack.

Read more: The Illustrated Transformer (multi-head) · Jay Alammar

Multi-Query Attention (MQA) inference

What: all query heads share one K and one V head (Shazeer, 2019). How: keep h query heads but a single K/V. Solves: shrinks the KV cache and memory-bandwidth pressure ~h×, which dominates decode latency. Cost: some quality loss and training instability.

Read more: Multi-Query Attention, explained · Tinkerd

Grouped-Query Attention (GQA) the sweet spot

What: interpolation between MHA and MQA (Ainslie, 2023) — G groups of query heads share a K/V head. How: G=h is MHA, G=1 is MQA; LLaMA-2/3, Mistral use G≈8. Solves: ~MQA inference cost with ~MHA quality; you can even uptrain an existing MHA checkpoint into GQA. This is the modern default.

Pros
  • Big KV-cache savings, tiny quality hit
  • Convertible from MHA
Cons
  • One more hyperparameter (group count)

Read more: Grouped-Query Attention · Sebastian Raschka

Multi-head Latent Attention (MLA) frontier

What: compress K/V into a low-rank latent vector (DeepSeek-V2/V3, 2024). How: instead of caching full K and V per head, cache a small shared latent that is projected up at use time (with a RoPE-compatible split). Solves: shrinks the KV cache even further than GQA while improving quality — a different axis (low-rank) than GQA’s head-sharing.

Read more: Multi-Head Latent Attention · Sebastian Raschka

FlashAttention how, not what

What: an exact attention algorithm that’s IO-aware (Dao, 2022; v2 2023; v3 2024). How: tile Q/K/V into blocks that fit in fast on-chip SRAM and use an online (streaming) softmax — running max and running sum — so the full n×n matrix is never materialized in slow HBM. Solves: the memory bottleneck — same math, O(n) memory, large wall-clock speedups; v3 adds FP8 and Hopper async. Crucially, it changes computation, not the result.

Pros
  • Exact result, much faster & lighter
  • Enables long-context training
Cons
  • Still O(n²) compute
  • Hardware-specific kernels

Read more: FlashAttention for long sequences · Stanford Hazy Research

Sliding-window / local attention long context

What: each token attends only to the last w tokens (Longformer 2020; Mistral 2023). How: a fixed local window; stacking L layers gives an effective receptive field of ~L·w. Solves: linear cost in n and a bounded (rolling) KV cache. Cost: sacrifices direct long-range links within a layer — often mixed with occasional full/global layers to recover them.

Read more: Longformer: local windowed attention · Hugging Face

Sparse attention long context

What: attend to a structured subset — local windows + a few global tokens + strided/random blocks (Longformer, BigBird), or learned/dynamic block selection (Native Sparse Attention, MoBA, 2025). How: a sparsity pattern (fixed or routed) cuts the score matrix. Solves: sub-quadratic long-context; the 2025 trainable variants get real speedups and keep quality, unlike older bolt-on sparsity.

Read more: Longformer: sparse attention · Hugging Face

Linear / kernel attention O(n)

What: replace softmax(QKᵀ)V with a kernel feature map so you can reorder the matmuls (Performer, Linear Transformers, Linformer, Reformer). How: compute φ(K)ᵀV first → O(n·d²) and constant-memory recurrence at decode. Solves: true linear complexity for very long sequences. Cost: the softmax approximation tends to underperform exact attention on hard recall/associative tasks.

Read more: Transformers are RNNs (linear attention) · project page

StreamingLLM / attention sinks streaming

What: keep a few initial “sink” tokens plus a rolling recent window (Xiao, 2023). How: models dump excess attention onto the first tokens (because softmax must sum to 1); preserving those “sinks” while sliding the rest lets generation run over effectively unbounded streams. Solves: the quality collapse when you naively evict the earliest tokens from the KV cache.

Read more: StreamingLLM & attention sinks · MIT Han Lab · GitHub

5. The math

Scaled dot-product attention

$$ \mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\left( \frac{QK^{\top}}{\sqrt{d_k}} + M \right) V $$

\(Q\in\mathbb{R}^{n\times d_k}\) queries, \(K\in\mathbb{R}^{n\times d_k}\) keys, \(V\in\mathbb{R}^{n\times d_v}\) values, n = sequence length. \(QK^{\top}\) is the \(n\times n\) score matrix (entry \(ij\) = relevance of token \(j\) to token \(i\)). \(M\) is the mask (\(0\) where allowed, \(-\infty\) where forbidden). Softmax is taken over each row.

Why divide by \(\sqrt{d_k}\)?

$$ q\cdot k = \sum_{i=1}^{d_k} q_i k_i \quad\Rightarrow\quad \mathrm{Var}(q\cdot k) = d_k $$

With unit-variance components the dot product has variance \(d_k\); for large head dims the logits get huge, softmax saturates into a near-one-hot spike, and gradients vanish. Dividing by \(\sqrt{d_k}\) normalizes the standard deviation back to \(1\). Omit it and deep models barely train.

Numerically stable (online) softmax

$$ \mathrm{softmax}(z)_i = \frac{e^{\,z_i - \max_j z_j}}{\sum_{k} e^{\,z_k - \max_j z_j}} $$

Subtracting the row max prevents overflow. The running max + running sum is exactly what FlashAttention computes incrementally so it never stores a full row.

Multi-head attention

$$ \mathrm{MHA}(x) = \big[\,\mathrm{head}_1 \,\Vert\, \cdots \,\Vert\, \mathrm{head}_h\,\big]\,W_O,\qquad \mathrm{head}_i = \mathrm{Attention}(xW_Q^i,\,xW_K^i,\,xW_V^i) $$

h heads, each of dim \(d_k=d/h\); \(\Vert\) is concatenation; \(W_O\in\mathbb{R}^{d\times d}\) mixes heads. Total Q/K/V/O parameters: \(4d^2\) (independent of \(h\)).

KV-cache size (the long-context memory wall)

$$ \text{KV bytes} = 2 \cdot L \cdot n_{kv} \cdot d_{\text{head}} \cdot n_{\text{seq}} \cdot b_{\text{batch}} \cdot \text{bytes} $$

Factor 2 = keys + values; L layers; \(n_{kv}\) = KV heads (← exactly what MQA/GQA/MLA shrink); \(n_{\text{seq}}\) = context length. It grows linearly with context and dominates inference memory.

6. In code

Scaled dot-product attention with a causal mask (PyTorch):

python
import torch
import torch.nn.functional as F

def attention(Q, K, V, mask=None):
    # Q, K, V: (batch, heads, seq, d_head)
    d_head = Q.size(-1)
    scores = (Q @ K.transpose(-2, -1)) / d_head ** 0.5      # (b, h, n, n)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))  # forbidden -> -inf
    weights = F.softmax(scores, dim=-1)                     # each query row sums to 1
    return weights @ V                                      # (b, h, n, d_head)

# causal mask: query i may attend only to keys j ≤ i
n = 5
causal = torch.tril(torch.ones(n, n)).bool()               # lower-triangular True

Splitting into heads — reshape (b, n, d) into (b, h, n, d/h):

python
def split_heads(x, heads):
    b, n, d = x.shape
    # (b, n, d) -> (b, n, h, d/h) -> (b, h, n, d/h) so each head attends in parallel
    return x.view(b, n, heads, d // heads).transpose(1, 2)

7. Worked example — attention by hand

Causal attention for the last token of “the cat sat.” “sat” issues a query and scores it against the keys of all tokens it can see (1, 2, 3). Suppose the scaled scores (after \(QK^\top/\sqrt{d_k}\)) come out as:

$$ s = [\,s_{\text{the}},\, s_{\text{cat}},\, s_{\text{sat}}\,] = [\,2.0,\; 1.0,\; 0.0\,] $$

Softmax turns scores into weights (using \(e^{2}=7.389,\ e^{1}=2.718,\ e^{0}=1\), sum \(=11.107\)):

$$ \alpha = \left[\tfrac{7.389}{11.107},\ \tfrac{2.718}{11.107},\ \tfrac{1.000}{11.107}\right] \approx [\,0.665,\; 0.245,\; 0.090\,] $$

Mix the values with \(v_{\text{the}}=[1,0]\), \(v_{\text{cat}}=[0,1]\), \(v_{\text{sat}}=[1,1]\):

$$ \mathbf{o} = 0.665\,[1,0] + 0.245\,[0,1] + 0.090\,[1,1] = [\,0.755,\; 0.335\,] $$

All three weights sum to 1, the masked future contributes nothing, and the output is a blend dominated by “the.” That vector replaces “sat”’s representation and flows on to the FFN and the next layer.

8. When to use what — choosing a variant

  • Always use a FlashAttention kernel — it’s exact, just faster, and orthogonal to the choices below.
  • Default modern stack → GQA (great quality/throughput balance); go MLA for the smallest KV cache at the frontier.
  • Need very long context cheaply → sliding-window (+ a few global/full layers) or sparse (MoBA / NSA).
  • Unbounded streaming chat → StreamingLLM sinks + rolling window.
  • Extreme length, willing to trade recall → linear/kernel attention or an SSM hybrid.

9. Common pitfalls & misconceptions

  • “The √dk is cosmetic.” Remove it and large-head models barely train — softmax saturates and gradients die.
  • “FlashAttention approximates attention.” It’s exact. It changes memory traffic, not the result. It’s also still O(n²) compute.
  • “Attention weights explain the model.” Contested — high attention ≠ causal importance; values and later layers matter, and attention can be manipulated without changing outputs.
  • “GQA/MQA save FLOPs.” Their main win is memory bandwidth / KV-cache size at decode, not raw compute.
  • “Forgetting the padding mask is harmless.” Unmasked PAD tokens leak into attention and the loss, silently degrading quality.

10. 🚀 What’s new in the industry

🚀 Fighting the n² and the KV-cache wall
  • Exact-but-faster. FlashAttention-2/3 push attention toward GPU peak FLOPs with better parallelism, async scheduling, and FP8 — long context became trainable, not just theoretical.
  • Shrinking the KV cache. GQA is now standard; MLA (DeepSeek-V2/V3) compresses K/V into a low-rank latent; YOCO caches KV essentially once across layers.
  • Trainable sparse attention. 2025’s Native Sparse Attention and MoBA make sparsity native to training (not a bolt-on), getting real long-context speedups without quality loss.

12. Key takeaways

  • Attention = softmax(QKᵀ/√d)·V — a soft, content-based, fully-parallel lookup with an O(1) path between any two tokens. The √d keeps softmax sane.
  • Multi-head lets one layer track many relationships at once; causal masking hides the future for generation.
  • The whole efficiency story is fighting n² and the KV cache: FlashAttention (exact, IO-aware), MQA/GQA/MLA (smaller cache), sliding-window/sparse/linear (sub-quadratic).
  • GQA is the modern default; MLA pushes the cache smaller still.
  • Attention sinks explain odd first-token attention and enable streaming.
