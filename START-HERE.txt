HOW TO USE THIS (for me)
------------------------
Each week, open a new agent IN THIS FOLDER (teaching-site/) and paste the
prompt below, replacing <TOPICS> with the topic(s) you're teaching that session
(one or several — comma-separated). The agent reads CLAUDE.md (the full build
spec) and builds a SEPARATE reveal.js slide DECK for each topic.

- First session ever: build "attention" or "tokenization" FIRST — the first
  deck sets the visual standard every later deck copies.
- The agent must leave already-built decks untouched and just add the new one(s)
  (and add a card to the index.html launcher).
- Everything is local + static: when done, double-click index.html, click a deck.
- While teaching: ←/→ to move, S = speaker notes, O = overview, F = fullscreen.


PASTE THIS TO THE AGENT  (fill in <TOPICS>)
-------------------------------------------
Read CLAUDE.md in this folder fully and follow it exactly. It is the complete
build spec for a local, static, no-build teaching SLIDE-DECK site built with
reveal.js (open by double-clicking index.html — no npm, no server, all libraries
via CDN; slides written as inline HTML <section> elements so file:// works).

Build a separate reveal.js slide DECK for EACH of these topics: <TOPICS>

Requirements:
- Build ONE separate, complete deck per topic — never merge topics into one deck.
  Add every new deck as a card on index.html.
- Follow the 15-section deck structure in CLAUDE.md, in order. Do not skip
  sections. Make each a real 30-45 minute lecture (~40-60 slides), not a summary.
  Put the deeper teaching detail in reveal.js speaker notes (<aside class="notes">).
- The "Approaches — full landscape" section is the heart of the deck — go DEEP:
  cover EVERY major approach/method/variant for this topic that matters for AI
  roles, classic → modern → SOTA, each in detail (how it works step by step, what
  it solves, pros/cons, when used, how it differs from the prior approach). Give
  EACH major approach its OWN animation/animated diagram — ideally animating the
  evolution A → B → C — plus the comparison-table cheat-sheet slide.
- EVERY deck MUST include animations + clear diagrams (see "Animations & diagrams"
  in CLAUDE.md): reveal.js fragments for step-by-step reveals, Mermaid/SVG
  diagrams, and at least one custom animated visualization of the core mechanism
  (Play/Pause/Step/Reset or fragment-driven), with prefers-reduced-motion handling.
- Pull the topic scope and the practice questions from the matching file in
  ../interview-prep/ (filenames are NN-topic-name.md) and from
  ../interview-prep/README.md. Questions only — do NOT use ../interview-prep/
  answer-keys/. Do not invent facts or fake paper links.
- Include the "What's new in the industry" section with real, recent papers
  (title, year, link).
- If this is the first deck: also create index.html (launcher) and the shared
  assets/deck.css + assets/deck.js. Otherwise, reuse them and just add the new
  deck(s) + a card on index.html. Leave other decks untouched.
- When done, confirm each deck opens correctly via file:// with no console errors
  (math, code, Mermaid, fragments, speaker notes all working), and run through the
  "Definition of done" checklist in CLAUDE.md.
